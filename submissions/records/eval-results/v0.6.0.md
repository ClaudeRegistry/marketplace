# Records Claude Plugin Eval Results v0.6.0

## Scope

Runtime-policy and operational privacy release:

- Add `plan-runtime.mjs` and `lib/runtime-policy.mjs` to choose between local
  Records CLI, configured-but-consent-gated external runtimes, and structural
  fallback.
- Add script-level privacy gates for FHIR URLs, FHIR server access, hosted
  validators, Records API use, terminology services, and package downloads.
- Add `doctor-packages.mjs` and `lib/package-doctor.mjs` for FHIR package-cache,
  declared dependency, profile-canonical, generated-artifact, Java, and mixed
  FHIR-version diagnostics.
- Extend `validate.mjs` so `/records:validate` includes `runtimePlan`,
  `privacyGate`, `packageDoctor`, and `runtimeAttempts` in its JSON output.
- Execute a detected local `records validate-file <target> --format json`
  adapter before structural fallback when the Records CLI is available and
  returns an `OperationOutcome`.
- Keep URL validation blocked pending consent; no URL fetch or FHIR server call
  is attempted by the local helper.
